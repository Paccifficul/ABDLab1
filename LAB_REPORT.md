# Лабораторная работа 1. Нормализация данных в модель снежинки

## Цель

Преобразовать исходные CSV-файлы с продажами товаров для домашних питомцев в аналитическую модель данных типа "снежинка": выделить таблицу фактов и связанные таблицы измерений.

## Исходные данные

В папке `исходные данные` находятся 10 CSV-файлов:

- `MOCK_DATA.csv`
- `MOCK_DATA (1).csv` ... `MOCK_DATA (9).csv`

Каждый файл содержит 1000 строк. После загрузки в staging-таблице `mock_data` должно быть 10000 строк.

## Выделенные сущности

Факт:

- `fact_sales` - продажи: дата продажи, покупатель, продавец, товар, магазин, поставщик, количество и сумма продажи.

Измерения:

- `dim_customer` - покупатели;
- `dim_seller` - продавцы;
- `dim_product` - товары;
- `dim_store` - магазины;
- `dim_supplier` - поставщики;
- `dim_date` - календарные даты;
- `dim_location`, `dim_city`, `dim_country` - нормализованные географические данные;
- `dim_pet`, `dim_pet_type`, `dim_pet_breed`, `dim_pet_category` - данные о питомцах;
- `dim_product_category`, `dim_brand` - категории и бренды товаров.

## Файлы решения

- `docker-compose.yml` - поднимает PostgreSQL 16 и подключает SQL-скрипты и CSV-файлы.
- `init.sql` - создает staging-таблицу `mock_data` и загружает 10 CSV-файлов.
- `ddl.sql` - создает таблицы модели снежинки.
- `dml.sql` - заполняет измерения и таблицу фактов из `mock_data`.
- `checks.sql` - запросы для проверки количества строк и базовой аналитики.

## Запуск

```bash
docker compose up -d
```

После инициализации база доступна со следующими параметрами (данные подключения к БД изменены в целях безопасности):

- host: `localhost`
- port: `5432`
- database: `abd_lab1`
- user: `pipipupu`
- password: `check`

Для повторной полной инициализации нужно удалить volume и запустить контейнер заново:

```bash
docker compose down -v
docker compose up -d
```

## Проверка

После запуска были выполнены запросы из `checks.sql`. Основная проверка:

```sql
SELECT count(*) FROM mock_data;
SELECT count(*) FROM fact_sales;
```

Оба запроса вернули `10000`.
